Tính năng cơ bản

OIADB là một thư viện Python wrapper cho Android Debug Bridge (ADB) với nhiều tính năng mạnh mẽ, giúp đơn giản hóa việc tương tác với thiết bị Android thông qua ADB.

Tự động cài đặt ADB

OIADB tự động phát hiện và cài đặt ADB nếu cần, không yêu cầu cài đặt thủ công. Thư viện cũng tự động cài đặt và quản lý oiadb-server.apk trên thiết bị Android của bạn.

Quản lý thiết bị

Dễ dàng liệt kê, kết nối và quản lý nhiều thiết bị Android. Hỗ trợ kết nối qua USB và mạng không dây.

# Liệt kê các thiết bị đã kết nối
devices = adb.get_devices()

# Kết nối thiết bị qua mạng
from oiadb.commands import connect
connect.connect_over_network("192.168.1.100", 5555)

Quản lý ứng dụng

Cài đặt, gỡ cài đặt, khởi động và quản lý ứng dụng Android một cách dễ dàng.

# Cài đặt ứng dụng
adb.install_app("/path/to/app.apk")

# Gỡ cài đặt ứng dụng
adb.uninstall_app("com.example.app")

# Khởi động ứng dụng
from oiadb.commands import apps
apps.start_app("com.example.app")

Tương tác với thiết bị

Thực hiện các thao tác như nhấn, vuốt, nhập văn bản và các thao tác khác trên thiết bị Android.

from oiadb.commands import interaction

# Nhấn vào tọa độ màn hình
interaction.tap(500, 500)

# Vuốt màn hình
interaction.swipe(100, 500, 600, 500)

# Nhập văn bản
interaction.text_input("Hello World")

# Nhấn phím Home
interaction.home()

Quản lý file

Đẩy và kéo file giữa máy tính và thiết bị Android.

from oiadb.commands import file_ops

# Đẩy file từ máy tính lên thiết bị
file_ops.push("/path/on/computer", "/path/on/device")

# Lấy file từ thiết bị về máy tính
file_ops.pull("/path/on/device", "/path/on/computer")

Lệnh ADB tùy chỉnh

Thực thi bất kỳ lệnh ADB tùy chỉnh nào thông qua giao diện Python đơn giản.

# Thực thi lệnh ADB tùy chỉnh
result = adb.run("shell dumpsys battery")
print(result)

Nhận diện hình ảnh

Một trong những tính năng nổi bật của OIADB là khả năng nhận diện và tương tác với các phần tử trên màn hình dựa trên hình ảnh mẫu, thay vì phải dựa vào tọa độ cố định.

Quy trình nhận diện hình ảnh

Quy trình nhận diện hình ảnh trong OIADB bao gồm các bước:

  1. Input Image: Hình ảnh mẫu cần tìm kiếm
  2. Screen Capture: Chụp màn hình thiết bị hiện tại
  3. Image Processing: Tiền xử lý hình ảnh (chuyển đổi grayscale, thay đổi kích thước, chuẩn hóa)
  4. Template Matching: Sử dụng thuật toán so khớp mẫu của OpenCV
  5. Match Detection: Xác định tọa độ của các vị trí khớp tốt nhất
  6. Device Interaction: Thực hiện các thao tác như nhấn, vuốt tại vị trí đã xác định

Các tính năng nhận diện hình ảnh

Tìm kiếm hình ảnh

Tìm kiếm hình ảnh mẫu trên màn hình thiết bị với nhiều tùy chọn tùy chỉnh.

from oiadb.commands import image_interaction

# Tìm hình ảnh đơn
result = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.8,  # Ngưỡng tương đồng (0.0 - 1.0)
    region=None,    # Vùng tìm kiếm (x, y, width, height), None = toàn màn hình
    scale_range=(0.8, 1.2),  # Phạm vi tỷ lệ để tìm kiếm
    scale_steps=5,  # Số bước tỷ lệ
    rotation_range=(0, 0),  # Phạm vi góc xoay (độ)
    rotation_steps=1,  # Số bước góc xoay
    use_gray=True,  # Sử dụng ảnh xám để tăng tốc độ
    use_canny=False  # Sử dụng phát hiện cạnh Canny cho hình ảnh mờ
)

if result:
    x, y, confidence = result
    print(f"Đã tìm thấy hình ảnh tại ({x}, {y}) với độ tương đồng {confidence}")
else:
    print("Không tìm thấy hình ảnh")

Tương tác dựa trên hình ảnh

Thực hiện các thao tác như nhấn, vuốt, kéo thả dựa trên vị trí của hình ảnh mẫu.

# Tìm và nhấp vào hình ảnh
image_interaction.tap_image("/path/to/button.png", threshold=0.8)

# Đợi hình ảnh xuất hiện và nhấp vào
image_interaction.wait_and_tap_image("/path/to/button.png", timeout=10)

# Kéo từ hình ảnh này đến hình ảnh khác
image_interaction.drag_image_to_image("/path/to/source.png", "/path/to/target.png")

Đợi và kiểm tra hình ảnh

Đợi hình ảnh xuất hiện hoặc biến mất, kiểm tra sự hiện diện của hình ảnh trên màn hình.

# Kiểm tra xem hình ảnh có xuất hiện không
if image_interaction.is_image_present("/path/to/element.png"):
    print("Hình ảnh xuất hiện trên màn hình")

# Đợi hình ảnh xuất hiện
image_interaction.wait_for_image("/path/to/button.png", timeout=10)

# Đợi hình ảnh biến mất
image_interaction.wait_until_image_disappears("/path/to/loading.png", timeout=30)

Tùy chỉnh nâng cao

Nhiều tùy chọn để xử lý các điều kiện màn hình khác nhau như phân giải cao/thấp, hình ảnh mờ/vỡ, hình ảnh xoay.

# Tìm trên màn hình phân giải cao
result_high_res = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.8,
    scale_range=(1.0, 1.5),  # Tỷ lệ lớn hơn cho màn hình phân giải cao
    scale_steps=5
)

# Tìm hình ảnh mờ
result_blurry = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.7,  # Ngưỡng thấp hơn cho hình ảnh mờ
    use_canny=True  # Sử dụng phát hiện cạnh Canny cho hình ảnh mờ
)

XML Dump & Trợ năng

Tính năng XML Dump cho phép truy xuất cấu trúc XML của giao diện người dùng Android thông qua một local server, giúp phân tích cấu trúc UI nhanh chóng và tương tác với ứng dụng Android thông qua API trợ năng.

Kiến trúc XML Dump

Kiến trúc của tính năng XML Dump bao gồm:

  • Local Server: HTTP server chạy trên thiết bị Android
  • XML Parser: Bộ phân tích XML với khả năng lọc nhanh
  • Parameter Handler: Xử lý các tham số truy vấn
  • API Endpoints: Các điểm cuối API (/get_xml, /find_elements, /accessibility_actions)
  • Accessibility API: Tích hợp với API trợ năng của Android

Quy trình xử lý

Quy trình xử lý của tính năng XML Dump bao gồm ba luồng chính:

  1. XML Dump Flow: Truy xuất và lọc cấu trúc XML
  2. Find Elements Flow: Tìm kiếm các phần tử UI dựa trên tiêu chí
  3. Accessibility Flow: Thực hiện các hành động trợ năng

Các tính năng XML Dump & Trợ năng

Khởi động XML Server

Khởi động local server trên thiết bị Android để cung cấp các API XML Dump.

from oiadb.commands import xml_dump

# Khởi động server trên cổng 8000
server, server_thread = xml_dump.start_xml_server(port=8000)

# Thiết lập port forwarding
xml_dump.setup_port_forwarding(device_port=8000, host_port=8000)

# Lấy địa chỉ IP của thiết bị
device_ip = xml_dump.get_device_ip()
print(f"XML dump available at: http://{device_ip}:8000/get_xml")

Lấy XML Dump

Truy xuất cấu trúc XML của giao diện người dùng hiện tại với các tùy chọn lọc.

# Lấy toàn bộ XML
xml_content = xml_dump.get_xml_dump()

# Lấy XML với bộ lọc
filtered_xml = xml_dump.get_xml_dump(
    resource_id="button_login",
    text_value="Login",
    threshold=0.7
)

# Sử dụng cache để tăng tốc độ
cached_xml = xml_dump.get_xml_dump(
    resource_id="button_login",
    use_cache=True
)

Tìm phần tử UI

Tìm kiếm các phần tử UI dựa trên nhiều tiêu chí như resource ID, text, class, v.v.

# Tìm tất cả các nút
buttons = xml_dump.find_elements_by_criteria({"class": "android.widget.Button"})

# Tìm phần tử với nhiều tiêu chí
login_buttons = xml_dump.find_elements_by_criteria({
    "class": "android.widget.Button",
    "value": "Login",
    "threshold": 0.7
})

Thực hiện hành động trợ năng

Thực hiện các hành động trợ năng như nhấp, nhập text, cuộn, v.v. thông qua API trợ năng của Android.

# Nhấp vào nút có text "Login"
xml_dump.perform_accessibility_action(
    "click", 
    {"value": "Login", "threshold": 0.7}
)

# Nhập text vào trường EditText
xml_dump.perform_accessibility_action(
    "setText", 
    {"class": "android.widget.EditText"}, 
    "Hello from XML dump!"
)

Hỗ trợ đa giao diện

Tương thích với nhiều loại giao diện từ các nhà sản xuất khác nhau như Stock Android, Samsung OneUI, Xiaomi MIUI, v.v.

from oiadb.commands import ui_compatibility

# Tìm nút đăng nhập trên bất kỳ giao diện nào
login_button = ui_compatibility.find_ui_element(
    element_type="button",
    text="Login"
)

# Nhấp vào nút
ui_compatibility.click_ui_element(
    element_type="button",
    text="Login"
)

Lưu ý

Để sử dụng đầy đủ tính năng XML Dump và trợ năng, bạn cần cấp quyền trợ năng cho ứng dụng:

  1. Bật dịch vụ trợ năng trên thiết bị Android: Vào Cài đặt > Trợ năng > Dịch vụ đã cài đặt
  2. Hoặc cấp quyền ADB để điều khiển trợ năng:
adb shell settings put secure enabled_accessibility_services com.android.uiautomator/com.google.android.marvin.talkback.TalkBackService
adb shell settings put secure accessibility_enabled 1

Để biết thêm chi tiết về tính năng XML dump và trợ năng, vui lòng xem Hướng dẫn XML Dump & Trợ năng.

Tự động phát hiện nền tảng

OIADB tự động phát hiện nền tảng và điều chỉnh hành vi phù hợp, đảm bảo trải nghiệm nhất quán trên tất cả các hệ điều hành.

from oiadb.utils import platform_utils

# Kiểm tra nền tảng hiện tại
current_platform = platform_utils.get_current_platform()
print(f"Nền tảng hiện tại: {current_platform}")

# Kiểm tra xem có phải là Windows không
if platform_utils.is_windows():
    print("Đang chạy trên Windows")

# Kiểm tra xem có phải là Termux không
if platform_utils.is_termux():
    print("Đang chạy trên Termux")

Kiến trúc thư viện

OIADB được tổ chức thành các module chính, mỗi module đảm nhận một chức năng cụ thể.

Cấu trúc module

MyADB

Lớp chính để tương tác với ADB, cung cấp các phương thức cơ bản như khởi tạo, kết nối thiết bị, thực thi lệnh ADB, v.v.

from oiadb import MyADB

adb = MyADB(device_id=None, cache_enabled=True, timeout=30, 
            adb_path=None, auto_start_server=True, auto_install_adb=True)

commands

Chứa các lệnh chuyên biệt như tương tác, quản lý ứng dụng, thao tác file, v.v.

  • app_info: Lấy thông tin ứng dụng
  • apps: Quản lý ứng dụng
  • basic: Các lệnh ADB cơ bản
  • connect: Kết nối thiết bị qua mạng
  • device_actions: Các hành động thiết bị
  • device_info: Thông tin thiết bị
  • file_ops: Thao tác file
  • interaction: Tương tác với thiết bị
  • image_interaction: Tương tác dựa trên nhận diện hình ảnh
  • xml_dump: Dump XML và tương tác qua trợ năng
  • logs: Xem log thiết bị
  • permissions: Quản lý quyền

utils

Các tiện ích hỗ trợ như quản lý nền tảng, xử lý hình ảnh, v.v.

  • platform_utils: Hỗ trợ đa nền tảng
  • adb_manager: Quản lý cài đặt và cấu hình ADB
  • advanced: Các tiện ích nâng cao
  • image_recognition: Nhận diện hình ảnh

exceptions

Định nghĩa các ngoại lệ đặc biệt của thư viện để xử lý lỗi một cách hiệu quả.

from oiadb.exceptions import ADBError, DeviceNotFoundError, 
                         TimeoutError, ImageNotFoundError
