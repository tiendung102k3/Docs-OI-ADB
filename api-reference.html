Lớp MyADB

Lớp chính để tương tác với ADB. Cung cấp các phương thức cơ bản để khởi tạo, kết nối thiết bị, thực thi lệnh ADB, và nhiều tính năng khác.

Khởi tạo

__init__(device_id=None, cache_enabled=True, timeout=30, adb_path=None, auto_start_server=True, auto_install_adb=True)

Khởi tạo đối tượng MyADB với các tùy chọn.

Tham số:
  • device_id (str, optional): ID của thiết bị cần kết nối. Mặc định là None (thiết bị đầu tiên được tìm thấy).
  • cache_enabled (bool, optional): Bật/tắt cache để tăng hiệu suất. Mặc định là True.
  • timeout (int, optional): Thời gian chờ tối đa cho các lệnh ADB (giây). Mặc định là 30.
  • adb_path (str, optional): Đường dẫn đến tệp thực thi ADB. Mặc định là None (tự động tìm).
  • auto_start_server (bool, optional): Tự động khởi động ADB server nếu chưa chạy. Mặc định là True.
  • auto_install_adb (bool, optional): Tự động cài đặt ADB nếu không tìm thấy. Mặc định là True.
Ví dụ:
from oiadb import MyADB

# Khởi tạo với thiết bị mặc định
adb = MyADB()

# Hoặc chỉ định thiết bị cụ thể
adb = MyADB(device_id="emulator-5554")

# Tắt tự động cài đặt ADB (nếu bạn đã cài đặt ADB)
adb = MyADB(auto_install_adb=False)

Phương thức

run(command)

Chạy lệnh ADB tùy chỉnh.

Tham số:
  • command (str): Lệnh ADB cần thực thi.
Trả về:
  • str: Kết quả của lệnh ADB.
Ví dụ:
# Thực thi lệnh ADB tùy chỉnh
result = adb.run("shell dumpsys battery")
print(result)

get_devices()

Liệt kê các thiết bị đã kết nối.

Trả về:
  • list: Danh sách các thiết bị đã kết nối.
Ví dụ:
# Liệt kê các thiết bị đã kết nối
devices = adb.get_devices()
print(devices)

reboot_device()

Khởi động lại thiết bị.

Ví dụ:
# Khởi động lại thiết bị
adb.reboot_device()

install_app(apk_path)

Cài đặt ứng dụng từ file APK.

Tham số:
  • apk_path (str): Đường dẫn đến file APK.
Ví dụ:
# Cài đặt ứng dụng
adb.install_app("/path/to/app.apk")

uninstall_app(package_name)

Gỡ cài đặt ứng dụng.

Tham số:
  • package_name (str): Tên package của ứng dụng.
Ví dụ:
# Gỡ cài đặt ứng dụng
adb.uninstall_app("com.example.app")

push_file(local_path, remote_path)

Đẩy file lên thiết bị.

Tham số:
  • local_path (str): Đường dẫn đến file trên máy tính.
  • remote_path (str): Đường dẫn đến vị trí lưu trên thiết bị.
Ví dụ:
# Đẩy file lên thiết bị
adb.push_file("/path/on/computer", "/path/on/device")

pull_file(remote_path, local_path)

Lấy file từ thiết bị.

Tham số:
  • remote_path (str): Đường dẫn đến file trên thiết bị.
  • local_path (str): Đường dẫn đến vị trí lưu trên máy tính.
Ví dụ:
# Lấy file từ thiết bị
adb.pull_file("/path/on/device", "/path/on/computer")

Lấy thông tin thiết bị.

Trả về:

Ví dụ:

device_info = adb.get_device_info()

Module commands

Module commands chứa các lệnh chuyên biệt để tương tác với thiết bị Android.

interaction

Module cung cấp các phương thức để tương tác với thiết bị Android thông qua tọa độ màn hình.

tap(x, y)

Nhấn vào tọa độ màn hình.

Tham số:
  • x (int): Tọa độ x.
  • y (int): Tọa độ y.
Ví dụ:
from oiadb.commands import interaction

# Nhấn vào tọa độ màn hình
interaction.tap(500, 500)

swipe(x1, y1, x2, y2, duration=300)

Vuốt màn hình từ điểm (x1, y1) đến điểm (x2, y2).

Tham số:
  • x1 (int): Tọa độ x bắt đầu.
  • y1 (int): Tọa độ y bắt đầu.
  • x2 (int): Tọa độ x kết thúc.
  • y2 (int): Tọa độ y kết thúc.
  • duration (int, optional): Thời gian vuốt (ms). Mặc định là 300.
Ví dụ:
# Vuốt màn hình
interaction.swipe(100, 500, 600, 500)

text_input(text)

Nhập văn bản.

Tham số:
  • text (str): Văn bản cần nhập.
Ví dụ:
# Nhập văn bản
interaction.text_input("Hello World")

home()

Nhấn phím Home.

Ví dụ:
# Nhấn phím Home
interaction.home()

back()

Nhấn phím Back.

Ví dụ:
# Nhấn phím Back
interaction.back()

long_press(x, y, duration=1000)

Nhấn giữ tại tọa độ màn hình.

Tham số:
  • x (int): Tọa độ x.
  • y (int): Tọa độ y.
  • duration (int, optional): Thời gian nhấn giữ (ms). Mặc định là 1000.
Ví dụ:
# Nhấn giữ tại tọa độ màn hình
interaction.long_press(500, 500, 2000)

image_interaction

Module cung cấp các phương thức để tương tác với thiết bị Android dựa trên nhận diện hình ảnh.

find_image(template_path, threshold=0.8, region=None, scale_range=(0.8, 1.2), scale_steps=5, rotation_range=(0, 0), rotation_steps=1, use_gray=True, use_canny=False)

Tìm hình ảnh mẫu trên màn hình thiết bị.

Tham số:
  • template_path (str): Đường dẫn đến hình ảnh mẫu.
  • threshold (float, optional): Ngưỡng tương đồng (0.0 - 1.0). Mặc định là 0.8.
  • region (tuple, optional): Vùng tìm kiếm (x, y, width, height). Mặc định là None (toàn màn hình).
  • scale_range (tuple, optional): Phạm vi tỷ lệ để tìm kiếm. Mặc định là (0.8, 1.2).
  • scale_steps (int, optional): Số bước tỷ lệ. Mặc định là 5.
  • rotation_range (tuple, optional): Phạm vi góc xoay (độ). Mặc định là (0, 0).
  • rotation_steps (int, optional): Số bước góc xoay. Mặc định là 1.
  • use_gray (bool, optional): Sử dụng ảnh xám để tăng tốc độ. Mặc định là True.
  • use_canny (bool, optional): Sử dụng phát hiện cạnh Canny cho hình ảnh mờ. Mặc định là False.
Trả về:
  • tuple: (x, y, confidence) nếu tìm thấy, None nếu không tìm thấy.
Ví dụ:
from oiadb.commands import image_interaction

# Tìm hình ảnh đơn
result = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.8
)

if result:
    x, y, confidence = result
    print(f"Đã tìm thấy hình ảnh tại ({x}, {y}) với độ tương đồng {confidence}")
else:
    print("Không tìm thấy hình ảnh")

tap_image(template_path, threshold=0.8, tap_offset=(0, 0))

Tìm và nhấp vào hình ảnh.

Tham số:
  • template_path (str): Đường dẫn đến hình ảnh mẫu.
  • threshold (float, optional): Ngưỡng tương đồng (0.0 - 1.0). Mặc định là 0.8.
  • tap_offset (tuple, optional): Độ lệch (dx, dy) khi nhấp so với tâm hình ảnh. Mặc định là (0, 0).
Trả về:
  • bool: True nếu thành công, False nếu không tìm thấy hình ảnh.
Ví dụ:
# Tìm và nhấp vào hình ảnh
success = image_interaction.tap_image("/path/to/button.png", threshold=0.8)

if success:
    print("Đã nhấp vào hình ảnh thành công")
else:
    print("Không tìm thấy hình ảnh để nhấp")

wait_and_tap_image(template_path, timeout=10, interval=0.5, threshold=0.8)

Đợi hình ảnh xuất hiện và nhấp vào.

Tham số:
  • template_path (str): Đường dẫn đến hình ảnh mẫu.
  • timeout (int, optional): Thời gian tối đa đợi (giây). Mặc định là 10.
  • interval (float, optional): Khoảng thời gian giữa các lần tìm kiếm (giây). Mặc định là 0.5.
  • threshold (float, optional): Ngưỡng tương đồng (0.0 - 1.0). Mặc định là 0.8.
Trả về:
  • bool: True nếu thành công, False nếu hình ảnh không xuất hiện trong thời gian chờ.
Ví dụ:
# Đợi hình ảnh xuất hiện và nhấp vào
success = image_interaction.wait_and_tap_image("/path/to/button.png", timeout=10)

if success:
    print("Đã đợi và nhấp vào hình ảnh thành công")
else:
    print("Hình ảnh không xuất hiện trong thời gian chờ")

is_image_present(template_path, threshold=0.8)

Kiểm tra xem hình ảnh có xuất hiện trên màn hình không.

Tham số:
  • template_path (str): Đường dẫn đến hình ảnh mẫu.
  • threshold (float, optional): Ngưỡng tương đồng (0.0 - 1.0). Mặc định là 0.8.
Trả về:
  • bool: True nếu hình ảnh xuất hiện, False nếu không.
Ví dụ:
# Kiểm tra xem hình ảnh có xuất hiện không
if image_interaction.is_image_present("/path/to/element.png"):
    print("Hình ảnh xuất hiện trên màn hình")
else:
    print("Hình ảnh không xuất hiện trên màn hình")

drag_image_to_image(source_template_path, target_template_path, duration=800, threshold=0.8)

Kéo từ hình ảnh nguồn đến hình ảnh đích.

Tham số:
  • source_template_path (str): Đường dẫn đến hình ảnh nguồn.
  • target_template_path (str): Đường dẫn đến hình ảnh đích.
  • duration (int, optional): Thời gian kéo (ms). Mặc định là 800.
  • threshold (float, optional): Ngưỡng tương đồng (0.0 - 1.0). Mặc định là 0.8.
Trả về:
  • bool: True nếu thành công, False nếu không tìm thấy một hoặc cả hai hình ảnh.
Ví dụ:
# Kéo từ hình ảnh này đến hình ảnh khác
success = image_interaction.drag_image_to_image("/path/to/source.png", "/path/to/target.png")

if success:
    print("Đã kéo thành công từ nguồn đến đích")
else:
    print("Không thể kéo vì không tìm thấy một hoặc cả hai hình ảnh")

xml_dump

Module cung cấp các phương thức để truy xuất cấu trúc XML của giao diện người dùng Android và tương tác thông qua API trợ năng.

start_xml_server(port=8000)

Khởi động local server trên thiết bị Android để cung cấp các API XML Dump.

Tham số:
  • port (int, optional): Cổng để chạy server. Mặc định là 8000.
Trả về:
  • tuple: (server, server_thread) - đối tượng server và thread chạy server.
Ví dụ:
from oiadb.commands import xml_dump

# Khởi động server trên cổng 8000
server, server_thread = xml_dump.start_xml_server(port=8000)

get_xml_dump(resource_id=None, text_value=None, content_desc=None, class_name=None, threshold=0.8, use_cache=False)

Lấy XML dump của giao diện hiện tại.

Tham số:
  • resource_id (str, optional): Lọc theo resource ID. Mặc định là None.
  • text_value (str, optional): Lọc theo giá trị text. Mặc định là None.
  • content_desc (str, optional): Lọc theo content description. Mặc định là None.
  • class_name (str, optional): Lọc theo class name. Mặc định là None.
  • threshold (float, optional): Ngưỡng tương đồng cho việc so khớp mờ. Mặc định là 0.8.
  • use_cache (bool, optional): Sử dụng cache để tăng tốc độ. Mặc định là False.
Trả về:
  • str: Nội dung XML.
Ví dụ:
# Lấy toàn bộ XML
xml_content = xml_dump.get_xml_dump()

# Lấy XML với bộ lọc
filtered_xml = xml_dump.get_xml_dump(
    resource_id="button_login",
    text_value="Login",
    threshold=0.7
)

find_elements_by_criteria(criteria)

Tìm các phần tử UI khớp với tiêu chí.

Tham số:
  • criteria (dict): Tiêu chí tìm kiếm, có thể bao gồm các khóa: "id", "value", "content_desc", "class", "threshold".
Trả về:
  • list: Danh sách các phần tử khớp với tiêu chí.
Ví dụ:
# Tìm tất cả các nút
buttons = xml_dump.find_elements_by_criteria({"class": "android.widget.Button"})

# Tìm phần tử với nhiều tiêu chí
login_buttons = xml_dump.find_elements_by_criteria({
    "class": "android.widget.Button",
    "value": "Login",
    "threshold": 0.7
})

perform_accessibility_action(action, criteria, value=None)

Thực hiện hành động trợ năng trên phần tử UI khớp với tiêu chí.

Tham số:
  • action (str): Hành động trợ năng (ví dụ: "click", "setText", "scroll").
  • criteria (dict): Tiêu chí để tìm phần tử.
  • value (str, optional): Giá trị cho hành động (ví dụ: text để nhập). Mặc định là None.
Trả về:
  • bool: True nếu thành công, False nếu không.
Ví dụ:
# Nhấp vào nút có text "Login"
xml_dump.perform_accessibility_action(
    "click", 
    {"value": "Login", "threshold": 0.7}
)

# Nhập text vào trường EditText
xml_dump.perform_accessibility_action(
    "setText", 
    {"class": "android.widget.EditText"}, 
    "Hello from XML dump!"
)

stop_xml_server(server)

Dừng XML server.

Tham số:
  • server: Đối tượng server được trả về từ start_xml_server().
Ví dụ:
# Dừng server khi hoàn tất
xml_dump.stop_xml_server(server)

Các lệnh khác

OIADB còn cung cấp nhiều module lệnh khác để tương tác với thiết bị Android.

  • app_info: Lấy thông tin ứng dụng
  • basic: Các lệnh ADB cơ bản
  • connect: Kết nối thiết bị qua mạng
  • device_actions: Các hành động thiết bị
  • device_info: Thông tin thiết bị
  • logs: Xem log thiết bị
  • permissions: Quản lý quyền

Xem Hướng dẫn để biết thêm chi tiết về cách sử dụng các lệnh này.

Module utils

Module utils cung cấp các tiện ích hỗ trợ cho thư viện OIADB.

platform_utils

Module cung cấp các phương thức để hỗ trợ đa nền tảng.

get_current_platform()

Lấy thông tin về nền tảng hiện tại.

Trả về:
  • str: Tên nền tảng ("windows", "linux", "macos", "termux").
Ví dụ:
from oiadb.utils import platform_utils

# Kiểm tra nền tảng hiện tại
current_platform = platform_utils.get_current_platform()
print(f"Nền tảng hiện tại: {current_platform}")

is_windows()

Kiểm tra xem có phải là Windows không.

Trả về:
  • bool: True nếu là Windows, False nếu không.
Ví dụ:
# Kiểm tra xem có phải là Windows không
if platform_utils.is_windows():
    print("Đang chạy trên Windows")

is_linux()

Kiểm tra xem có phải là Linux không.

Trả về:
  • bool: True nếu là Linux, False nếu không.

is_macos()

Kiểm tra xem có phải là macOS không.

Trả về:
  • bool: True nếu là macOS, False nếu không.

is_termux()

Kiểm tra xem có phải là Termux không.

Trả về:
  • bool: True nếu là Termux, False nếu không.

adb_manager

Module cung cấp các phương thức để quản lý cài đặt và cấu hình ADB.

find_adb()

Tìm đường dẫn đến tệp thực thi ADB.

Trả về:
  • str: Đường dẫn đến tệp thực thi ADB, hoặc None nếu không tìm thấy.

install_adb()

Cài đặt ADB nếu chưa có.

Trả về:
  • str: Đường dẫn đến tệp thực thi ADB sau khi cài đặt.

start_adb_server()

Khởi động ADB server.

kill_adb_server()

Dừng ADB server.

image_recognition

Module cung cấp các phương thức cấp thấp để nhận diện hình ảnh.

find_template(screenshot, template, threshold=0.8, use_gray=True, use_canny=False)

Tìm hình ảnh mẫu trong ảnh chụp màn hình.

Tham số:
  • screenshot (numpy.ndarray): Ảnh chụp màn hình.
  • template (numpy.ndarray): Hình ảnh mẫu.
  • threshold (float, optional): Ngưỡng tương đồng (0.0 - 1.0). Mặc định là 0.8.
  • use_gray (bool, optional): Sử dụng ảnh xám để tăng tốc độ. Mặc định là True.
  • use_canny (bool, optional): Sử dụng phát hiện cạnh Canny cho hình ảnh mờ. Mặc định là False.
Trả về:
  • tuple: (x, y, confidence) nếu tìm thấy, None nếu không tìm thấy.

find_all_templates(screenshot, template, threshold=0.8, use_gray=True, use_canny=False)

Tìm tất cả các vị trí của hình ảnh mẫu trong ảnh chụp màn hình.

Tham số:
  • screenshot (numpy.ndarray): Ảnh chụp màn hình.
  • template (numpy.ndarray): Hình ảnh mẫu.
  • threshold (float, optional): Ngưỡng tương đồng (0.0 - 1.0). Mặc định là 0.8.
  • use_gray (bool, optional): Sử dụng ảnh xám để tăng tốc độ. Mặc định là True.
  • use_canny (bool, optional): Sử dụng phát hiện cạnh Canny cho hình ảnh mờ. Mặc định là False.
Trả về:
  • list: Danh sách các tuple (x, y, confidence).

scale_template(template, scale_factor)

Thay đổi kích thước hình ảnh mẫu.

Tham số:
  • template (numpy.ndarray): Hình ảnh mẫu.
  • scale_factor (float): Hệ số tỷ lệ.
Trả về:
  • numpy.ndarray: Hình ảnh mẫu đã thay đổi kích thước.

rotate_template(template, angle)

Xoay hình ảnh mẫu.

Tham số:
  • template (numpy.ndarray): Hình ảnh mẫu.
  • angle (float): Góc xoay (độ).
Trả về:
  • numpy.ndarray: Hình ảnh mẫu đã xoay.

Module exceptions

Module exceptions định nghĩa các ngoại lệ đặc biệt của thư viện OIADB.

ADBError

Ngoại lệ cơ bản cho các lỗi liên quan đến ADB.

Ví dụ:
from oiadb.exceptions import ADBError

try:
    # Thực hiện một số thao tác ADB
    pass
except ADBError as e:
    print(f"Lỗi ADB: {e}")

DeviceNotFoundError

Ngoại lệ khi không tìm thấy thiết bị.

Ví dụ:
from oiadb.exceptions import DeviceNotFoundError

try:
    # Thực hiện một số thao tác với thiết bị
    pass
except DeviceNotFoundError as e:
    print(f"Không tìm thấy thiết bị: {e}")

TimeoutError

Ngoại lệ khi thao tác vượt quá thời gian chờ.

Ví dụ:
from oiadb.exceptions import TimeoutError

try:
    # Thực hiện một số thao tác có thể mất nhiều thời gian
    pass
except TimeoutError as e:
    print(f"Thao tác vượt quá thời gian chờ: {e}")

ImageNotFoundError

Ngoại lệ khi không tìm thấy hình ảnh.

Ví dụ:
from oiadb.exceptions import ImageNotFoundError

try:
    # Thực hiện một số thao tác nhận diện hình ảnh
    pass
except ImageNotFoundError as e:
    print(f"Không tìm thấy hình ảnh: {e}")

OIADB
