Hướng dẫn nhận diện hình ảnh

Tính năng nhận diện hình ảnh của OIADB cho phép bạn tương tác với thiết bị Android dựa trên hình ảnh mẫu, thay vì phải dựa vào tọa độ cố định. Hướng dẫn này sẽ giúp bạn hiểu và sử dụng hiệu quả tính năng này.

Chuẩn bị hình ảnh mẫu

Để đạt được kết quả tốt nhất, hãy tuân thủ các nguyên tắc sau khi chuẩn bị hình ảnh mẫu:

Cắt chính xác

Cắt hình ảnh mẫu chỉ bao gồm phần tử cần tìm, loại bỏ các phần không liên quan. Tránh cắt quá sát hoặc quá rộng.

Kích thước phù hợp

Sử dụng hình ảnh mẫu có kích thước tương đối so với kích thước thực tế trên màn hình. Nếu phần tử quá nhỏ, hãy chụp màn hình ở độ phân giải cao hơn.

Độ tương phản cao

Chọn hình ảnh mẫu có độ tương phản cao với nền. Điều này giúp thuật toán nhận diện dễ dàng hơn.

Định dạng phù hợp

Lưu hình ảnh mẫu ở định dạng PNG để tránh mất chất lượng do nén. Tránh sử dụng JPEG cho các biểu tượng hoặc văn bản.

Tìm kiếm hình ảnh cơ bản

Dưới đây là cách sử dụng các phương thức tìm kiếm hình ảnh cơ bản:

from oiadb import MyADB
from oiadb.commands import image_interaction

# Khởi tạo ADB
adb = MyADB()

# Tìm hình ảnh đơn
result = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.8  # Ngưỡng tương đồng (0.0 - 1.0)
)

if result:
    x, y, confidence = result
    print(f"Đã tìm thấy hình ảnh tại ({x}, {y}) với độ tương đồng {confidence}")
    
    # Nhấp vào vị trí tìm thấy
    from oiadb.commands import interaction
    interaction.tap(x, y)
else:
    print("Không tìm thấy hình ảnh")

Tìm kiếm hình ảnh nâng cao

Để xử lý các trường hợp phức tạp hơn, bạn có thể sử dụng các tùy chọn nâng cao:

# Tìm hình ảnh với nhiều tùy chọn
result = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.8,
    region=(100, 100, 500, 500),  # Vùng tìm kiếm (x, y, width, height)
    scale_range=(0.8, 1.2),  # Phạm vi tỷ lệ để tìm kiếm
    scale_steps=5,  # Số bước tỷ lệ
    rotation_range=(-10, 10),  # Phạm vi góc xoay (độ)
    rotation_steps=5,  # Số bước góc xoay
    use_gray=True,  # Sử dụng ảnh xám để tăng tốc độ
    use_canny=False  # Sử dụng phát hiện cạnh Canny cho hình ảnh mờ
)

Giải thích các tham số

  • threshold: Ngưỡng tương đồng (0.0 - 1.0). Giá trị cao hơn yêu cầu khớp chính xác hơn, giá trị thấp hơn cho phép khớp mờ hơn.
  • region: Giới hạn vùng tìm kiếm để tăng tốc độ và giảm khả năng khớp sai.
  • scale_range: Phạm vi tỷ lệ để tìm kiếm. Hữu ích khi kích thước phần tử có thể thay đổi (ví dụ: do độ phân giải màn hình khác nhau).
  • rotation_range: Phạm vi góc xoay để tìm kiếm. Hữu ích khi phần tử có thể bị xoay.
  • use_gray: Sử dụng ảnh xám để tăng tốc độ. Nên bật trừ khi màu sắc là yếu tố quan trọng để phân biệt.
  • use_canny: Sử dụng phát hiện cạnh Canny. Hữu ích cho hình ảnh mờ hoặc có độ tương phản thấp.

Tương tác dựa trên hình ảnh

OIADB cung cấp nhiều phương thức để tương tác với thiết bị dựa trên hình ảnh:

# Tìm và nhấp vào hình ảnh
image_interaction.tap_image("/path/to/button.png", threshold=0.8)

# Đợi hình ảnh xuất hiện và nhấp vào
image_interaction.wait_and_tap_image(
    "/path/to/button.png", 
    timeout=10,  # Thời gian tối đa đợi (giây)
    interval=0.5  # Khoảng thời gian giữa các lần tìm kiếm (giây)
)

# Kéo từ hình ảnh này đến hình ảnh khác
image_interaction.drag_image_to_image(
    "/path/to/source.png", 
    "/path/to/target.png",
    duration=800  # Thời gian kéo (ms)
)

# Đợi hình ảnh xuất hiện
image_interaction.wait_for_image(
    "/path/to/button.png", 
    timeout=10
)

# Đợi hình ảnh biến mất
image_interaction.wait_until_image_disappears(
    "/path/to/loading.png", 
    timeout=30
)

Xử lý các trường hợp đặc biệt

Màn hình phân giải cao

Đối với màn hình phân giải cao, hãy sử dụng phạm vi tỷ lệ lớn hơn:

result_high_res = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.8,
    scale_range=(1.0, 1.5),  # Tỷ lệ lớn hơn cho màn hình phân giải cao
    scale_steps=5
)

Hình ảnh mờ

Đối với hình ảnh mờ hoặc có độ tương phản thấp, hãy sử dụng phát hiện cạnh Canny:

result_blurry = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.7,  # Ngưỡng thấp hơn cho hình ảnh mờ
    use_canny=True  # Sử dụng phát hiện cạnh Canny
)

Hình ảnh xoay

Đối với hình ảnh có thể bị xoay, hãy sử dụng phạm vi góc xoay:

result_rotated = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.8,
    rotation_range=(-30, 30),  # Phạm vi góc xoay ±30 độ
    rotation_steps=10  # Số bước góc xoay
)

Hình ảnh màu

Đối với hình ảnh mà màu sắc là yếu tố quan trọng để phân biệt:

result_color = image_interaction.find_image(
    template_path="/path/to/template.png",
    threshold=0.8,
    use_gray=False  # Sử dụng ảnh màu thay vì ảnh xám
)

Mẹo và thủ thuật

  • Sử dụng vùng tìm kiếm: Giới hạn vùng tìm kiếm để tăng tốc độ và giảm khả năng khớp sai.
  • Điều chỉnh ngưỡng: Bắt đầu với ngưỡng cao (0.9) và giảm dần nếu không tìm thấy hình ảnh.
  • Tạo nhiều mẫu: Đối với phần tử có nhiều trạng thái (ví dụ: nút bật/tắt), hãy tạo mẫu cho mỗi trạng thái.
  • Xử lý ngoại lệ: Luôn kiểm tra kết quả trả về và xử lý trường hợp không tìm thấy hình ảnh.
  • Tối ưu hiệu suất: Sử dụng use_gray=True và vùng tìm kiếm hợp lý để tăng tốc độ.

Hướng dẫn XML Dump & Trợ năng

Tính năng XML Dump của OIADB cho phép bạn truy xuất cấu trúc XML của giao diện người dùng Android và tương tác với ứng dụng thông qua API trợ năng. Hướng dẫn này sẽ giúp bạn hiểu và sử dụng hiệu quả tính năng này.

Thiết lập XML Server

Trước khi sử dụng tính năng XML Dump, bạn cần thiết lập XML Server trên thiết bị Android:

from oiadb import MyADB
from oiadb.commands import xml_dump

# Khởi tạo ADB
adb = MyADB()

# Khởi động server trên cổng 8000
server, server_thread = xml_dump.start_xml_server(port=8000)

# Thiết lập port forwarding
xml_dump.setup_port_forwarding(device_port=8000, host_port=8000)

# Lấy địa chỉ IP của thiết bị
device_ip = xml_dump.get_device_ip()
print(f"XML dump available at: http://{device_ip}:8000/get_xml")

Lưu ý

XML Server sẽ tự động cài đặt và khởi động oiadb-server.apk trên thiết bị Android của bạn. Đảm bảo thiết bị đã bật chế độ USB Debugging và đã cấp quyền trợ năng.

Lấy XML Dump

Sau khi thiết lập XML Server, bạn có thể lấy XML dump của giao diện hiện tại:

# Lấy toàn bộ XML
xml_content = xml_dump.get_xml_dump()
print(xml_content)

# Lấy XML với bộ lọc
filtered_xml = xml_dump.get_xml_dump(
    resource_id="button_login",  # Lọc theo resource ID
    text_value="Login",  # Lọc theo giá trị text
    threshold=0.7  # Ngưỡng tương đồng cho việc so khớp mờ
)

# Sử dụng cache để tăng tốc độ
cached_xml = xml_dump.get_xml_dump(
    resource_id="button_login",
    use_cache=True  # Sử dụng cache
)

Tìm phần tử UI

Bạn có thể tìm các phần tử UI dựa trên nhiều tiêu chí khác nhau:

# Tìm tất cả các nút
buttons = xml_dump.find_elements_by_criteria({"class": "android.widget.Button"})
for button in buttons:
    print(f"Button: {button.get('text')}, Bounds: {button.get('bounds')}")

# Tìm phần tử với nhiều tiêu chí
login_buttons = xml_dump.find_elements_by_criteria({
    "class": "android.widget.Button",
    "value": "Login",
    "threshold": 0.7
})

# Tìm phần tử theo ID
id_elements = xml_dump.find_elements_by_criteria({
    "id": "com.example.app:id/submit_button"
})

# Tìm phần tử theo content description
content_desc_elements = xml_dump.find_elements_by_criteria({
    "content_desc": "Submit form"
})

Thực hiện hành động trợ năng

Bạn có thể thực hiện các hành động trợ năng trên các phần tử UI:

# Nhấp vào nút có text "Login"
xml_dump.perform_accessibility_action(
    "click", 
    {"value": "Login", "threshold": 0.7}
)

# Nhập text vào trường EditText
xml_dump.perform_accessibility_action(
    "setText", 
    {"class": "android.widget.EditText"}, 
    "Hello from XML dump!"
)

# Cuộn danh sách
xml_dump.perform_accessibility_action(
    "scroll", 
    {"class": "android.widget.ListView"}
)

# Nhấn giữ phần tử
xml_dump.perform_accessibility_action(
    "longClick", 
    {"id": "com.example.app:id/item"}
)

OIADB hỗ trợ nhiều hành động trợ năng khác nhau:

Phân tích cấu trúc XML

Bạn có thể phân tích cấu trúc XML để hiểu rõ hơn về giao diện người dùng:

import xml.etree.ElementTree as ET

# Lấy XML dump
xml_content = xml_dump.get_xml_dump()

# Phân tích XML
root = ET.fromstring(xml_content)

# Duyệt qua tất cả các phần tử
for element in root.findall(".//*"):
    # Lấy thuộc tính của phần tử
    class_name = element.get("class")
    text = element.get("text")
    resource_id = element.get("resource-id")
    content_desc = element.get("content-desc")
    bounds = element.get("bounds")
    
    # In thông tin phần tử
    if text:
        print(f"Element: {class_name}, Text: {text}, ID: {resource_id}")

Mẹo và thủ thuật

  • Sử dụng cache: Bật cache để tăng tốc độ khi thực hiện nhiều thao tác trên cùng một màn hình.
  • Kết hợp với nhận diện hình ảnh: Kết hợp XML Dump với nhận diện hình ảnh để xử lý các trường hợp phức tạp.
  • Xử lý ngoại lệ: Luôn kiểm tra kết quả trả về và xử lý trường hợp không tìm thấy phần tử.
  • Sử dụng ngưỡng tương đồng: Điều chỉnh ngưỡng tương đồng để xử lý các trường hợp text không khớp chính xác.
  • Dừng server khi hoàn tất: Dừng XML server khi không cần sử dụng nữa để giải phóng tài nguyên.
# Dừng server khi hoàn tất
xml_dump.stop_xml_server(server)

Kết nối không dây

OIADB hỗ trợ kết nối với thiết bị Android qua mạng không dây, giúp bạn tương tác với thiết bị mà không cần kết nối USB. Hướng dẫn này sẽ giúp bạn thiết lập và sử dụng kết nối không dây.

Thiết lập kết nối không dây

Để thiết lập kết nối không dây, bạn cần thực hiện các bước sau:

from oiadb import MyADB
from oiadb.commands import connect

# Khởi tạo ADB
adb = MyADB()

# Bước 1: Kết nối thiết bị qua USB
# Đảm bảo thiết bị đã được kết nối qua USB và đã bật USB Debugging

# Bước 2: Lấy địa chỉ IP của thiết bị
device_ip = connect.get_device_ip()
print(f"Device IP: {device_ip}")

# Bước 3: Thiết lập kết nối không dây
connect.enable_wireless_debugging(port=5555)

# Bước 4: Ngắt kết nối USB và kết nối qua mạng
# Bây giờ bạn có thể ngắt kết nối USB
connect.connect_over_network(device_ip, 5555)

# Kiểm tra kết nối
devices = adb.get_devices()
print(devices)

Lưu ý

Để sử dụng kết nối không dây, thiết bị Android và máy tính phải nằm trong cùng một mạng. Đảm bảo tường lửa không chặn kết nối.

Kết nối tự động

Bạn có thể tự động kết nối với thiết bị đã biết địa chỉ IP:

# Kết nối tự động với thiết bị đã biết địa chỉ IP
def auto_connect_to_device(ip, port=5555, max_retries=3):
    from oiadb.commands import connect
    import time
    
    for i in range(max_retries):
        try:
            connect.connect_over_network(ip, port)
            print(f"Connected to {ip}:{port}")
            return True
        except Exception as e:
            print(f"Attempt {i+1}/{max_retries} failed: {e}")
            time.sleep(2)
    
    return False

# Sử dụng
auto_connect_to_device("192.168.1.100")

Quản lý nhiều thiết bị không dây

Nếu bạn cần quản lý nhiều thiết bị không dây, bạn có thể sử dụng mã sau:

# Quản lý nhiều thiết bị không dây
def manage_wireless_devices(device_list):
    from oiadb import MyADB
    from oiadb.commands import connect
    
    connected_devices = []
    
    for device in device_list:
        ip = device["ip"]
        port = device.get("port", 5555)
        name = device.get("name", ip)
        
        try:
            connect.connect_over_network(ip, port)
            device_id = f"{ip}:{port}"
            adb = MyADB(device_id=device_id)
            connected_devices.append({
                "name": name,
                "ip": ip,
                "port": port,
                "device_id": device_id,
                "adb": adb
            })
            print(f"Connected to {name} ({ip}:{port})")
        except Exception as e:
            print(f"Failed to connect to {name} ({ip}:{port}): {e}")
    
    return connected_devices

# Sử dụng
devices = [
    {"name": "Phone 1", "ip": "192.168.1.100"},
    {"name": "Tablet", "ip": "192.168.1.101", "port": 5556}
]

connected_devices = manage_wireless_devices(devices)

# Sử dụng thiết bị đã kết nối
for device in connected_devices:
    adb = device["adb"]
    name = device["name"]
    print(f"Working with {name}")
    # Thực hiện các thao tác với thiết bị
    device_info = adb.get_device_info()
    print(device_info)

Mẹo và thủ thuật

  • Địa chỉ IP tĩnh: Cấu hình thiết bị Android với địa chỉ IP tĩnh để dễ dàng kết nối lại.
  • Kết nối lại tự động: Triển khai cơ chế kết nối lại tự động khi mất kết nối.
  • Kiểm tra kết nối: Thường xuyên kiểm tra trạng thái kết nối để đảm bảo thiết bị vẫn được kết nối.
  • Sử dụng ADB Pair: Đối với Android 11 trở lên, sử dụng tính năng ADB Pair để kết nối an toàn hơn.
# Kiểm tra kết nối
def check_device_connection(device_id):
    from oiadb import MyADB
    
    try:
        adb = MyADB(device_id=device_id)
        devices = adb.get_devices()
        return device_id in devices
    except:
        return False

# Sử dụng
is_connected = check_device_connection("192.168.1.100:5555")
print(f"Device connected: {is_connected}")

Tự động hóa ứng dụng

OIADB cho phép bạn tự động hóa các tác vụ trên ứng dụng Android. Hướng dẫn này sẽ giúp bạn xây dựng các kịch bản tự động hóa hiệu quả.

Cài đặt và khởi động ứng dụng

Bước đầu tiên trong tự động hóa ứng dụng là cài đặt và khởi động ứng dụng:

from oiadb import MyADB
from oiadb.commands import apps

# Khởi tạo ADB
adb = MyADB()

# Cài đặt ứng dụng
adb.install_app("/path/to/app.apk")

# Lấy thông tin package và activity chính
from oiadb.commands import app_info
package_info = app_info.get_package_info("/path/to/app.apk")
package_name = package_info["package_name"]
main_activity = package_info["main_activity"]

# Khởi động ứng dụng
apps.start_app(package_name)

# Hoặc khởi động ứng dụng với activity cụ thể
apps.start_activity(f"{package_name}/{main_activity}")

Xây dựng kịch bản tự động hóa

Bạn có thể kết hợp nhiều phương thức để xây dựng kịch bản tự động hóa phức tạp:

# Ví dụ: Tự động đăng nhập vào ứng dụng
def auto_login(username, password):
    from oiadb.commands import interaction, image_interaction
    import time
    
    # Khởi động ứng dụng
    apps.start_app("com.example.app")
    time.sleep(2)  # Đợi ứng dụng khởi động
    
    # Phương pháp 1: Sử dụng nhận diện hình ảnh
    if image_interaction.is_image_present("/path/to/username_field.png"):
        # Nhấp vào trường username
        image_interaction.tap_image("/path/to/username_field.png")
        # Nhập username
        interaction.text_input(username)
        
        # Nhấp vào trường password
        image_interaction.tap_image("/path/to/password_field.png")
        # Nhập password
        interaction.text_input(password)
        
        # Nhấp vào nút đăng nhập
        image_interaction.tap_image("/path/to/login_button.png")
    
    # Phương pháp 2: Sử dụng XML Dump
    else:
        from oiadb.commands import xml_dump
        
        # Nhập username
        xml_dump.perform_accessibility_action(
            "setText", 
            {"class": "android.widget.EditText", "hint": "Username"}, 
            username
        )
        
        # Nhập password
        xml_dump.perform_accessibility_action(
            "setText", 
            {"class": "android.widget.EditText", "hint": "Password"}, 
            password
        )
        
        # Nhấp vào nút đăng nhập
        xml_dump.perform_accessibility_action(
            "click", 
            {"class": "android.widget.Button", "value": "Login"}
        )
    
    # Đợi màn hình chính xuất hiện
    return image_interaction.wait_for_image("/path/to/home_screen.png", timeout=10)

Xử lý các trường hợp đặc biệt

Trong quá trình tự động hóa, bạn có thể gặp phải các trường hợp đặc biệt cần xử lý:

Xử lý hộp thoại

Xử lý các hộp thoại xuất hiện trong quá trình tự động hóa:

def handle_dialog():
    from oiadb.commands import image_interaction
    
    # Kiểm tra các loại hộp thoại phổ biến
    if image_interaction.is_image_present("/path/to/permission_dialog.png"):
        # Nhấp vào nút "Allow"
        image_interaction.tap_image("/path/to/allow_button.png")
        return True
    
    if image_interaction.is_image_present("/path/to/update_dialog.png"):
        # Nhấp vào nút "Later"
        image_interaction.tap_image("/path/to/later_button.png")
        return True
    
    # Sử dụng XML Dump để xử lý hộp thoại không nhận diện được
    from oiadb.commands import xml_dump
    
    # Tìm và nhấp vào nút "OK" hoặc "Allow"
    for button_text in ["OK", "Allow", "Accept", "Yes", "Continue"]:
        try:
            xml_dump.perform_accessibility_action(
                "click", 
                {"class": "android.widget.Button", "value": button_text}
            )
            return True
        except:
            pass
    
    return False

Xử lý màn hình loading

Đợi màn hình loading biến mất trước khi tiếp tục:

def wait_for_loading_to_finish(timeout=30):
    from oiadb.commands import image_interaction
    import time
    
    start_time = time.time()
    
    while time.time() - start_time < timeout:
        if not image_interaction.is_image_present("/path/to/loading_indicator.png"):
            # Đợi thêm một chút để đảm bảo màn hình đã tải xong
            time.sleep(1)
            return True
        
        # Đợi một chút trước khi kiểm tra lại
        time.sleep(0.5)
    
    return False

Thử lại thao tác

Thử lại thao tác nếu không thành công:

def retry_operation(operation_func, max_retries=3, *args, **kwargs):
    import time
    
    for i in range(max_retries):
        try:
            result = operation_func(*args, **kwargs)
            if result:
                return True
        except Exception as e:
            print(f"Attempt {i+1}/{max_retries} failed: {e}")
        
        # Đợi một chút trước khi thử lại
        time.sleep(1)
    
    return False

Mẹo và thủ thuật

  • Kết hợp nhiều phương pháp: Kết hợp nhận diện hình ảnh và XML Dump để xử lý các trường hợp phức tạp.
  • Sử dụng thời gian chờ: Thêm thời gian chờ hợp lý giữa các thao tác để đảm bảo ứng dụng có đủ thời gian phản hồi.
  • Xử lý ngoại lệ: Luôn xử lý các ngoại lệ có thể xảy ra trong quá trình tự động hóa.
  • Kiểm tra trạng thái: Thường xuyên kiểm tra trạng thái của ứng dụng để đảm bảo kịch bản đang chạy đúng.
  • Ghi log: Ghi log các thao tác và kết quả để dễ dàng gỡ lỗi.

Tối ưu hiệu suất

Khi làm việc với OIADB, việc tối ưu hiệu suất là rất quan trọng, đặc biệt là khi bạn cần tự động hóa các tác vụ phức tạp hoặc làm việc với nhiều thiết bị. Hướng dẫn này sẽ giúp bạn tối ưu hiệu suất của OIADB.

Sử dụng cache

OIADB cung cấp cơ chế cache để tăng tốc độ thực thi các lệnh lặp lại:

from oiadb import MyADB

# Khởi tạo ADB với cache bật
adb = MyADB(cache_enabled=True)

# Sử dụng cache cho XML Dump
from oiadb.commands import xml_dump
xml_content = xml_dump.get_xml_dump(use_cache=True)

# Xóa cache khi cần
adb.clear_cache()

Tối ưu nhận diện hình ảnh

Nhận diện hình ảnh có thể tốn nhiều tài nguyên. Dưới đây là một số cách để tối ưu:

from oiadb.commands import image_interaction

# Sử dụng vùng tìm kiếm
result = image_interaction.find_image(
    template_path="/path/to/template.png",
    region=(100, 100, 500, 500)  # Giới hạn vùng tìm kiếm
)

# Sử dụng ảnh xám để tăng tốc độ
result = image_interaction.find_image(
    template_path="/path/to/template.png",
    use_gray=True  # Mặc định là True
)

# Giảm số bước tỷ lệ và góc xoay
result = image_interaction.find_image(
    template_path="/path/to/template.png",
    scale_range=(0.9, 1.1),  # Phạm vi tỷ lệ nhỏ hơn
    scale_steps=3,  # Ít bước tỷ lệ hơn
    rotation_range=(0, 0)  # Không xoay
)

Tối ưu XML Dump

XML Dump cũng có thể được tối ưu để tăng hiệu suất:

from oiadb.commands import xml_dump

# Sử dụng bộ lọc để giảm kích thước XML
filtered_xml = xml_dump.get_xml_dump(
    class_name="android.widget.Button"  # Chỉ lấy các nút
)

# Sử dụng cache
cached_xml = xml_dump.get_xml_dump(
    use_cache=True
)

# Tối ưu tìm kiếm phần tử
buttons = xml_dump.find_elements_by_criteria({
    "class": "android.widget.Button",
    "visible": True  # Chỉ lấy các phần tử hiển thị
})

Tối ưu thời gian chờ

Điều chỉnh thời gian chờ phù hợp để tránh lãng phí thời gian:

from oiadb import MyADB

# Điều chỉnh thời gian chờ mặc định

# Điều chỉnh thời gian chờ cho các thao tác cụ thể
from oiadb.commands import image_interaction

image_interaction.is_image_present("/path/to/optional_element.png", timeout=1)

# Tăng thời gian chờ cho các thao tác quan trọng

Tối ưu kết nối

Quản lý bộ nhớ hiệu quả để tránh rò rỉ bộ nhớ:

# Xóa các biến lớn khi không cần thiết

# ...
# Xóa biến khi không cần thiết nữa

# Sử dụng context manager để tự động giải phóng tài nguyên

    def __enter__(self):

        return self.xml_content

        del self.xml_content
        gc.collect()

# Sử dụng

    # Xử lý XML
    # ...

Mẹo và thủ thuật

  • Sử dụng đa luồng: Sử dụng đa luồng để thực hiện các thao tác độc lập đồng thời.
  • Tránh polling liên tục: Sử dụng các phương thức đợi thay vì kiểm tra liên tục.
  • Tối ưu hóa hình ảnh mẫu: Sử dụng hình ảnh mẫu nhỏ và đơn giản để tăng tốc độ nhận diện.
  • Sử dụng phương thức phù hợp: Chọn phương thức phù hợp với từng tác vụ (ví dụ: XML Dump cho tìm kiếm phần tử, nhận diện hình ảnh cho xác nhận trực quan).
  • Giám sát hiệu suất: Sử dụng các công cụ giám sát hiệu suất để xác định các điểm nghẽn.

Xử lý sự cố nâng cao

Khi làm việc với OIADB, bạn có thể gặp phải các sự cố phức tạp. Hướng dẫn này sẽ giúp bạn xử lý các sự cố nâng cao.

Xử lý lỗi kết nối

Lỗi kết nối là một trong những sự cố phổ biến nhất khi làm việc với ADB:

from oiadb import MyADB
from oiadb.exceptions import DeviceNotFoundError, ADBError

def handle_connection_issues():
    try:
        # Thử khởi tạo ADB
        adb = MyADB()
        return adb
    except DeviceNotFoundError:
        # Không tìm thấy thiết bị
        print("Device not found. Trying to reconnect...")
        
        # Khởi động lại ADB server
        from oiadb.utils import adb_manager
        adb_manager.kill_adb_server()
        adb_manager.start_adb_server()
        
        # Thử kết nối lại
        try:
            adb = MyADB()
            return adb
        except DeviceNotFoundError:
            # Vẫn không tìm thấy thiết bị
            print("Device still not found. Please check USB connection.")
            
            # Thử kết nối không dây
            try:
                from oiadb.commands import connect
                device_ip = input("Enter device IP: ")
                connect.connect_over_network(device_ip, 5555)
                adb = MyADB(device_id=f"{device_ip}:5555")
                return adb
            except Exception as e:
                print(f"Wireless connection failed: {e}")
                return None
    except ADBError as e:
        print(f"ADB error: {e}")
        return None

# Sử dụng
adb = handle_connection_issues()
if adb:
    print("Connection established")
    # Tiếp tục với các thao tác khác
else:
    print("Failed to establish connection")

Xử lý lỗi nhận diện hình ảnh

Nhận diện hình ảnh có thể gặp sự cố do nhiều yếu tố khác nhau:

from oiadb.commands import image_interaction
from oiadb.exceptions import ImageNotFoundError
import os
import time

def handle_image_recognition_issues(template_path, max_retries=3):
    # Kiểm tra xem file hình ảnh mẫu có tồn tại không
    if not os.path.exists(template_path):
        print(f"Template image not found: {template_path}")
        return None
    
    # Thử với các ngưỡng khác nhau
    thresholds = [0.8, 0.7, 0.6]
    
    for threshold in thresholds:
        for i in range(max_retries):
            try:
                result = image_interaction.find_image(
                    template_path=template_path,
                    threshold=threshold
                )
                if result:
                    print(f"Image found with threshold {threshold}")
                    return result
            except ImageNotFoundError:
                pass
            
            # Đợi một chút trước khi thử lại
            time.sleep(1)
    
    # Thử với các tùy chọn nâng cao
    try:
        # Thử với ảnh xám
        result = image_interaction.find_image(
            template_path=template_path,
            threshold=0.6,
            use_gray=True
        )
        if result:
            print("Image found with grayscale")
            return result
        
        # Thử với phát hiện cạnh Canny
        result = image_interaction.find_image(
            template_path=template_path,
            threshold=0.6,
            use_canny=True
        )
        if result:
            print("Image found with Canny edge detection")
            return result
        
        # Thử với phạm vi tỷ lệ rộng hơn
        result = image_interaction.find_image(
            template_path=template_path,
            threshold=0.6,
            scale_range=(0.5, 1.5),
            scale_steps=10
        )
        if result:
            print("Image found with wider scale range")
            return result
    except Exception as e:
        print(f"Advanced image recognition failed: {e}")
    
    print("Image recognition failed with all attempts")
    return None

# Sử dụng
result = handle_image_recognition_issues("/path/to/template.png")
if result:
    x, y, confidence = result
    print(f"Image found at ({x}, {y}) with confidence {confidence}")
else:
    print("Image not found")

Xử lý lỗi XML Dump

XML Dump có thể gặp sự cố do nhiều yếu tố khác nhau:

from oiadb.commands import xml_dump
import xml.etree.ElementTree as ET
import time

def handle_xml_dump_issues(max_retries=3):
    for i in range(max_retries):
        try:
            # Thử lấy XML dump
            xml_content = xml_dump.get_xml_dump()
            
            # Kiểm tra xem XML có hợp lệ không
            try:
                root = ET.fromstring(xml_content)
                return xml_content
            except ET.ParseError:
                print(f"Invalid XML (attempt {i+1}/{max_retries})")
        except Exception as e:
            print(f"XML dump failed (attempt {i+1}/{max_retries}): {e}")
        
        # Đợi một chút trước khi thử lại
        time.sleep(2)
    
    # Thử khởi động lại XML server
    try:
        print("Restarting XML server...")
        server, server_thread = xml_dump.start_xml_server(port=8000)
        xml_dump.setup_port_forwarding(device_port=8000, host_port=8000)
        
        # Thử lại sau khi khởi động lại server
        time.sleep(2)
        xml_content = xml_dump.get_xml_dump()
        
        try:
            root = ET.fromstring(xml_content)
            return xml_content
        except ET.ParseError:
            print("Invalid XML after server restart")
    except Exception as e:
        print(f"XML server restart failed: {e}")
    
    print("XML dump failed with all attempts")
    return None

# Sử dụng
xml_content = handle_xml_dump_issues()
if xml_content:
    print("XML dump successful")
    # Tiếp tục xử lý XML
else:
    print("XML dump failed")

Xử lý lỗi ứng dụng

Ứng dụng Android có thể gặp sự cố trong quá trình tự động hóa:

from oiadb.commands import apps, device_actions
import time

def handle_app_issues(package_name):
    # Kiểm tra xem ứng dụng có đang chạy không
    from oiadb.commands import app_info
    running_apps = app_info.get_running_apps()
    
    if package_name not in running_apps:
        print(f"App {package_name} is not running. Starting...")
        apps.start_app(package_name)
        time.sleep(2)
    
    # Kiểm tra xem ứng dụng có bị treo không
    is_app_responsive = check_app_responsiveness(package_name)
    
    if not is_app_responsive:
        print(f"App {package_name} is not responsive. Restarting...")
        # Dừng ứng dụng
        apps.stop_app(package_name)
        time.sleep(1)
        
        # Khởi động lại ứng dụng
        apps.start_app(package_name)
        time.sleep(2)
        
        # Kiểm tra lại
        is_app_responsive = check_app_responsiveness(package_name)
        
        if not is_app_responsive:
            print(f"App {package_name} is still not responsive. Clearing data...")
            # Xóa dữ liệu ứng dụng
            apps.clear_app_data(package_name)
            time.sleep(1)
            
            # Khởi động lại ứng dụng
            apps.start_app(package_name)
            time.sleep(2)
            
            # Kiểm tra lại
            is_app_responsive = check_app_responsiveness(package_name)
            
            if not is_app_responsive:
                print(f"App {package_name} is still not responsive. Rebooting device...")
                # Khởi động lại thiết bị
                device_actions.reboot_device()
                time.sleep(30)  # Đợi thiết bị khởi động lại
                
                # Khởi động lại ứng dụng
                apps.start_app(package_name)
                time.sleep(2)
                
                # Kiểm tra lại
                is_app_responsive = check_app_responsiveness(package_name)
                
                if not is_app_responsive:
                    print(f"App {package_name} is still not responsive after all attempts")
                    return False
    
    return True

def check_app_responsiveness(package_name):
    # Kiểm tra xem ứng dụng có phản hồi không
    try:
        # Thử thực hiện một thao tác đơn giản
        from oiadb.commands import interaction
        interaction.tap(100, 100)  # Nhấp vào một vị trí bất kỳ
        
        # Kiểm tra xem ứng dụng có còn chạy không
        from oiadb.commands import app_info
        running_apps = app_info.get_running_apps()
        
        return package_name in running_apps
    except:
        return False

# Sử dụng
app_ok = handle_app_issues("com.example.app")
if app_ok:
    print("App is running and responsive")
    # Tiếp tục với các thao tác khác
else:
    print("Failed to handle app issues")

  • Thử lại với các tùy chọn khác nhau: Thử lại với các tùy chọn khác nhau khi gặp sự cố.
  • Kiểm tra điều kiện trước khi thực hiện: Kiểm tra điều kiện trước khi thực hiện các thao tác để tránh lỗi.
